  # go generate
  used to run cmd befor running your code
  go generate ./...
  It scans your code for special comments and executes them.
  ex. Inside go file
  //go:generate <command>
  
  //go:generate echo "Hello, Code Generation!"
  
  cmd:
  go generate
  ✔ Output:
  
  Hello, Code Generation!
  
  Works as a comment (ignored by compiler)
  Runs commands in shell
  File must be in a Go package

ex. Mock Genration for testing
isnide repository,
package user

type UserRepository interface {
    GetUser(id int) (string, error)
}

keep this cmd in the repo file:
//go:generate mockgen -source=repo.go -destination=mock_repo.go -package=user

then when we hit:
go generate ./...

it will create
// mock_repo.go (auto-generated)
type MockUserRepository struct {
    ctrl *gomock.Controller
}

then if we write test cases, then whis will use this mockRepsotiry
